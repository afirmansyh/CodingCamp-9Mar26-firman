🔍 Accessibility Testing Guide

✅ Keyboard Navigation Test

Instructions: Open index.html and test the following:

  1. Tab Key: Press Tab repeatedly to navigate through all interactive elements
    • Timer Start → Stop → Reset buttons
    • Task input field → Add button
    • Task checkboxes → Edit → Delete buttons (for each task)
    • Link name input → Link URL input → Add Link button
    • Link buttons → Delete buttons (for each link)
  2. Shift+Tab: Navigate backwards through elements
  3. Enter Key:
    • Type in task input and press Enter to add task
    • Focus on any button and press Enter to activate
  4. Space Key:
    • Focus on buttons and press Space to activate
    • Focus on checkboxes and press Space to toggle

✓ All elements should be reachable via keyboard

✓ Focus indicators should be clearly visible (blue outline)

✅ ARIA Labels Test

Instructions: Use browser DevTools to inspect elements:

  • Right-click any element → Inspect
  • Check for aria-label, aria-labelledby, or role attributes

Expected ARIA attributes:

  • Timer display: role="timer", aria-live="polite"
  • Timer buttons: aria-label="Start timer", etc.
  • Task checkboxes: aria-label="Mark task ... as complete"
  • Task buttons: aria-label="Edit task ...", aria-label="Delete task ..."
  • Links: aria-label="... (opens in new tab)"

✓ All interactive elements have descriptive labels

✅ Screen Reader Test

Instructions: Test with a screen reader:

  • Windows: NVDA (free) or JAWS
  • Mac: VoiceOver (Cmd+F5)
  • Chrome: ChromeVox extension

What to test:

  1. Navigate through the page and verify all elements are announced
  2. Add a task and verify the screen reader announces the new task
  3. Toggle a task checkbox and verify status change is announced
  4. Add a link and verify it's announced
  5. Verify timer updates are announced (aria-live regions)

✓ Screen readers should announce all content and changes

✅ Color Contrast Test

Instructions: Use browser DevTools or online tools:

  • Chrome DevTools: Inspect element → Styles → Click color swatch → View contrast ratio
  • Online: WebAIM Contrast Checker

Expected contrast ratios (WCAG AA):

  • Normal text: 4.5:1 minimum
  • Large text (18pt+): 3:1 minimum

Actual ratios:

  • Text (#333) on white: 12.6:1 ✓ (AAA)
  • Accent (#4a90e2) on white: 3.4:1 ✓ (AA for large text)
  • Completed (#767676) on white: 4.5:1 ✓ (AA)

✅ Focus Indicators Test

Instructions: Tab through all elements and verify:

  • All buttons show a visible blue outline when focused
  • All inputs show a blue border and subtle shadow when focused
  • All links show a visible outline when focused
  • Focus indicators have sufficient contrast (visible against background)

✓ Focus indicators use 2px solid outline with 2px offset

✅ Semantic HTML Test

Instructions: View page source or inspect with DevTools:

  • <main> wraps main content
  • <section> for each component
  • <h2> for component headings
  • <ul> and <li> for task list
  • <button> for all interactive controls
  • <input type="checkbox"> for task completion
  • <input type="url"> for link URLs

✓ Proper semantic HTML elements used throughout
